# 🧵 Threading & UI Safety Summary (v1.08.9.8.3)

**Last Updated:** 2026-02-09  
**Applies To:** Zeta Download Manager + Patient UI  
**Status:** ✅ Standardized

---

## ✅ Current Threading Rules

1. **Never create or modify Qt widgets from worker threads.**
2. **UI updates must execute on the Qt main thread** using one of:
   - Qt signals/slots
   - `QTimer.singleShot(...)`
3. **Background work** should run in worker threads or executors and only emit signals for UI updates.

---

## 📌 Where This Matters Most

- **Download UI:** `PacsClient/zeta_download_manager/ui/main_widget.py`
- **Patient UI:** `PacsClient/pacs/patient_tab/ui/patient_ui/patient_widget.py`

If you touch these files, confirm UI updates are still main‑thread‑safe.

---

## ✅ Expected Behavior

- No `QObject::setParent` errors
- Download UI remains responsive
- Progress updates are smooth

---

## 🧠 AI Notes (Explicit Guidance)

- If you add UI callbacks inside worker threads, **wrap them** or **emit signals**.
- Do **not** directly create `QWidget`, `QDialog`, or `QPixmap` in background threads.
- Always re‑validate thread safety after touching `zeta_download_manager` UI classes.
