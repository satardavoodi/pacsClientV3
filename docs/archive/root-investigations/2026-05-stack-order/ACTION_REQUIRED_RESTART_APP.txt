### Technical Root Cause (For Documentation)
The matrix M[2,3] controls the display-to-raw k-coordinate transformation:
- **Before:** M[2,3] = 0.0 → raw_k = display_k (identity, 0-based to 0-based)
- **After:** M[2,3] = -1.0 → raw_k = display_k - 1 (conversion, 1-based to 0-based)

This implements the design contract that DisplayGeometry uses 1-based display indices.
