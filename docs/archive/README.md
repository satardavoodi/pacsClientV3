# Archive

Archived documents are kept for historical traceability, not as the primary source of truth.

## Subdirectories

- `module-framework/`: superseded module execution planning and delivery docs
- `root-guides/`: one-off implementation guides and older engineering notes that were previously stored in the repository root
- `../releases/archive/`: version-by-version release notes and commit message artifacts

## Reading Archived Notes Safely

- Expect stale paths, version numbers, and implementation assumptions.
- Prefer the documents in `docs/architecture/`, `docs/modules/`, and `docs/development/` first.
- Treat archive content as background context, not current architecture.
