## Rules

1. **Every fix gets a row.** No exceptions. The PR that lands the fix adds the row in the same diff.
2. **Module column lists every subsystem touched.** A bug that crossed `download_manager` and `viewer` is two rows in the Module cell.
3. **Guard test must FAIL on the pre-fix codebase** and PASS on the fixed one. Without that property, the row is decorative.
4. **Cross-module bugs go under `tests/code/system/test_<date>_<slug>.py`.** Single-module bugs live next to the module under `tests/code/<module>/test_<short>_regression.py`.
5. **Don't remove rows.** If a guard becomes obsolete (the surface it protects was deleted), mark the row with `(retired YYYY-MM-DD)` and keep it for archaeology.
