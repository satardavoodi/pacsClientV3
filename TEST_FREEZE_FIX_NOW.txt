---

WHAT WAS THE PROBLEM?

User reported: "App freezes when downloading"

Root cause analysis found:
- Every downloaded image (1000+ per study) triggers a progress update
- Each update calls state_store.update() immediately
- Each update triggers UIObserver → ui.update_progress_bar()
- Each UI update defers with QTimer.singleShot(0, ...)
- Result: 1000+ deferred callbacks queued in event loop
- Event loop becomes overloaded → Main thread frozen

The app appeared to hang for 5-10 minutes, even though the download was only supposed to take 2-3 minutes.

---

WHAT WAS THE FIX?

Implemented progress callback THROTTLING:

1. Instead of updating on EVERY image, batch updates
2. Update every 100ms instead of 1000+ times per download
3. Result: ~10 updates per download instead of 1000+
4. 100x reduction in event loop pressure
