💡 HOW THE FIX WORKS

  Problem: 1000+ progress callbacks per download flooding event loop

  Solution: Batch callbacks every 100ms

  Before:
    Image 1 → Update (1%)
    Image 2 → Update (2%)
    Image 3 → Update (3%)
    ...
    Image 1000 → Update (100%)
    = 1000 updates in event loop = FROZEN ❌

  After:
    Images 1-10 → Batch (10%)
    Images 11-20 → Batch (20%)
    Images 21-30 → Batch (30%)
    ...
    = ~10 updates in event loop = SMOOTH ✅
