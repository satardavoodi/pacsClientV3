=== THREADING FIX SUMMARY ===
Date: February 5, 2026
Component: Zeta Download Manager UI
Severity: CRITICAL (App freezing issue)
Status: ✅ RESOLVED

PROBLEM:
--------
- 30+ "QObject::setParent" errors in logs
- UI widget creation happening in worker threads instead of main Qt thread
- Application completely frozen during downloads
- Unresponsive interface, unable to interact with Download Manager

ROOT CAUSE:
-----------
UIObserver class was calling UI update methods directly from worker threads:
- update_progress_bar()
- update_status_badge()
- update_priority_badge()
- update_action_buttons()
- update_current_series()
- refresh_table_order()

When Qt widgets are modified from non-main threads, Qt emits error:
"QObject::setParent: Cannot set parent, new parent is in a different thread"

SOLUTION APPLIED:
-----------------
Wrapped all 6 UI update methods with QTimer.singleShot(0, ...)

This defers widget operations to the main Qt thread instead of executing 
directly in worker thread.

Example pattern:
```python
def update_progress_bar(self, study_uid: str, progress: float) -> None:
    """Called by UIObserver (may be in worker thread)"""
    QTimer.singleShot(0, lambda: self._do_update_progress_bar(study_uid, progress))

def _do_update_progress_bar(self, study_uid: str, progress: float) -> None:
    """Actually updates widget (runs in main thread)"""
    # Safe widget operations here ✓
```
